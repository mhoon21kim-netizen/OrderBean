# OlderBean 사용자 화면 UI/UX PRD
## User Interface Product Requirements Document

## 1. 개요 (Overview)

본 문서는 **OlderBean** 커피 주문 서비스의 **사용자 화면(User Interface)**에 대한 상세한 UI/UX 요구사항을 정의합니다. 사용자가 메뉴를 조회하고, 옵션을 선택하여 주문을 완료할 수 있는 직관적이고 효율적인 인터페이스를 제공하는 것을 목표로 합니다.

### 목표

- **빠른 주문**: 15초 이내 주문 완료 가능한 직관적인 UI
- **명확한 정보 전달**: 메뉴 정보, 가격, 옵션이 한눈에 보이는 구조
- **편리한 장바구니**: 실시간 가격 계산 및 주문 관리
- **반응형 디자인**: 모바일 및 데스크톱 환경 모두 지원

---

## 2. 화면 구조 (Screen Structure)

### 2.1 전체 레이아웃

```
┌─────────────────────────────────────────┐
│           헤더 영역 (Header)              │
│  [COZY 로고]    [주문하기] [관리자]        │
├─────────────────────────────────────────┤
│                                         │
│         메뉴 표시 영역 (Menu Grid)        │
│  ┌─────┐  ┌─────┐  ┌─────┐              │
│  │카드1│  │카드2│  │카드3│              │
│  └─────┘  └─────┘  └─────┘              │
│                                         │
├─────────────────────────────────────────┤
│         장바구니 영역 (Shopping Cart)     │
│  ┌───────────────────────────────────┐  │
│  │ 장바구니                          │  │
│  │ - 메뉴1 (옵션) X 1   4,500원      │  │
│  │ - 메뉴2 X 2           8,000원      │  │
│  │ 총 금액: 12,500원                 │  │
│  │ [주문하기]                         │  │
│  └───────────────────────────────────┘  │
└─────────────────────────────────────────┘
```

### 2.2 화면 구성 요소

1. **헤더 (Header)**
2. **메뉴 그리드 (Menu Grid)**
3. **장바구니 (Shopping Cart)**

## 3. 상세 컴포넌트 명세

### 3.1 헤더 영역 (Header)

#### 3.1.1 레이아웃

```
┌────────────────────────────────────────────┐
│  COZY    [주문하기]  [관리자]              │
└────────────────────────────────────────────┘
```

#### 3.1.2 구성 요소

| 요소 | 설명 | 스타일 |
|------|------|--------|
| **로고** | "COZY" 텍스트 또는 이미지 로고 | 왼쪽 정렬, 브랜드 색상 |
| **주문하기 버튼** | 주문 내역 페이지로 이동 | 오른쪽 정렬, 버튼 스타일 |
| **관리자 버튼** | 관리자 페이지로 이동 (권한 필요) | 오른쪽 정렬, 버튼 스타일 |

#### 3.1.3 기능 요구사항

- **로고 클릭**: 홈 화면으로 이동
- **주문하기 버튼**: 
  - 클릭 시 주문 내역 페이지로 이동
  - 로그인 상태 확인 필요
  - 비로그인 시 로그인 페이지로 리다이렉트
- **관리자 버튼**:
  - 관리자 권한이 있는 경우에만 표시
  - 클릭 시 관리자 대시보드로 이동

#### 3.1.4 디자인 요구사항

- **높이**: 60-80px
- **배경색**: 흰색 또는 브랜드 색상
- **테두리**: 하단에 얇은 구분선 (선택)
- **반응형**: 모바일에서는 햄버거 메뉴로 변경 가능

### 3.2 메뉴 표시 영역 (Menu Grid)

#### 3.2.1 레이아웃

```
┌─────────────┐  ┌─────────────┐  ┌─────────────┐
│  메뉴 카드 1  │  │  메뉴 카드 2  │  │  메뉴 카드 3  │
│             │  │             │  │             │
│  [이미지]    │  │  [이미지]    │  │  [이미지]    │
│  메뉴명      │  │  메뉴명      │  │  메뉴명      │
│  가격        │  │  가격        │  │  가격        │
│  설명        │  │  설명        │  │  설명        │
│  [옵션]      │  │  [옵션]      │  │  [옵션]      │
│  [담기]      │  │  [담기]      │  │  [담기]      │
└─────────────┘  └─────────────┘  └─────────────┘
```

#### 3.2.2 메뉴 카드 구성 요소

| 요소 | 설명 | 필수 여부 |
|------|------|----------|
| **메뉴 이미지** | 커피 메뉴 사진 | 선택 (플레이스홀더 제공) |
| **메뉴명** | 메뉴 이름 (예: "아메리카노(ICE)") | 필수 |
| **가격** | 기본 가격 (예: "4,000원") | 필수 |
| **설명** | 메뉴에 대한 간단한 설명 | 선택 |
| **옵션 체크박스** | 추가 옵션 선택 (샷 추가, 시럽 추가 등) | 선택 |
| **담기 버튼** | 장바구니에 추가 | 필수 |

#### 3.2.3 메뉴 카드 상세 명세

##### 3.2.3.1 메뉴 이미지
- **크기**: 카드 너비에 맞춤, 비율 유지 (16:9 또는 1:1)
- **플레이스홀더**: 이미지가 없는 경우 기본 이미지 표시
- **로딩**: 이미지 로딩 중 스켈레톤 UI 표시
- **에러 처리**: 이미지 로드 실패 시 플레이스홀더 표시

##### 3.2.3.2 메뉴명
- **표시 형식**: 메뉴명 (온도/타입)
  - 예: "아메리카노(ICE)", "아메리카노(HOT)", "카페라떼"
- **스타일**: 굵은 글씨, 카드 상단에 위치
- **길이 제한**: 긴 메뉴명은 말줄임표 처리

##### 3.2.3.3 가격
- **표시 형식**: 숫자 + "원" (예: "4,000원")
- **천 단위 구분**: 콤마로 구분
- **스타일**: 강조 색상, 메뉴명 아래 위치
- **실시간 업데이트**: 옵션 선택 시 가격 자동 업데이트

##### 3.2.3.4 설명
- **내용**: 메뉴에 대한 간단한 설명
- **길이 제한**: 1-2줄, 초과 시 말줄임표 처리
- **스타일**: 작은 글씨, 회색 계열
- **선택 사항**: 설명이 없는 경우 표시하지 않음

##### 3.2.3.5 옵션 체크박스
- **옵션 종류**:
  - 샷 추가 (+500원)
  - 시럽 추가 (+0원 또는 +가격)
  - 기타 커스터마이징 옵션
- **표시 형식**: 체크박스 + 옵션명 + 추가 가격
- **동작**:
  - 체크 시: 옵션 선택, 가격에 추가
  - 체크 해제 시: 옵션 제거, 가격에서 차감
- **가격 업데이트**: 옵션 선택/해제 시 카드 내 가격 실시간 업데이트

##### 3.2.3.6 담기 버튼
- **위치**: 카드 하단
- **스타일**: 주요 액션 버튼 (브랜드 색상)
- **동작**:
  - 클릭 시: 선택된 옵션과 함께 장바구니에 추가
  - 성공 피드백: 토스트 메시지 또는 버튼 애니메이션
  - 장바구니 아이콘 업데이트: 장바구니 아이템 수 표시

#### 3.2.4 그리드 레이아웃

- **데스크톱**: 3열 그리드 (최대 3개 카드 가로 배치)
- **태블릿**: 2열 그리드
- **모바일**: 1열 그리드 (세로 스크롤)
- **간격**: 카드 간 적절한 여백 (16-24px)
- **스크롤**: 메뉴가 많을 경우 세로 스크롤

#### 3.2.5 기능 요구사항

- **메뉴 목록 조회**: 
  - 페이지 로드 시 자동으로 메뉴 목록 API 호출
  - 로딩 상태 표시 (스켈레톤 UI)
  - 에러 처리 (에러 메시지 표시)
- **옵션 선택**:
  - 체크박스로 옵션 선택/해제
  - 선택된 옵션은 시각적으로 강조
  - 가격 실시간 계산 및 표시
- **장바구니 추가**:
  - "담기" 버튼 클릭 시 장바구니에 추가
  - 중복 메뉴 추가 시 수량 증가 또는 별도 항목으로 추가 (설정 가능)
  - 성공 피드백 제공

### 3.3 장바구니 영역 (Shopping Cart)

#### 3.3.1 레이아웃

```
┌──────────────────────────────────────┐
│ 장바구니                              │
├──────────────────────────────────────┤
│ • 아메리카노(ICE) (샷 추가) X 1       │
│   4,500원                            │
│                                      │
│ • 아메리카노(HOT) X 2                 │
│   8,000원                            │
├──────────────────────────────────────┤
│ 총 금액: 12,500원                    │
│                                      │
│        [주문하기]                     │
└──────────────────────────────────────┘
```

#### 3.3.2 구성 요소

| 요소 | 설명 | 필수 여부 |
|------|------|----------|
| **장바구니 헤더** | "장바구니" 제목 | 필수 |
| **주문 항목 목록** | 장바구니에 담긴 메뉴 목록 | 필수 |
| **총 금액** | 모든 항목의 합계 가격 | 필수 |
| **주문하기 버튼** | 주문 완료 버튼 | 필수 |

#### 3.3.3 상세 명세

##### 3.3.3.1 장바구니 헤더
- **텍스트**: "장바구니"
- **스타일**: 굵은 글씨, 영역 상단에 위치
- **아이콘**: 장바구니 아이콘 추가 가능 (선택)

##### 3.3.3.2 주문 항목 목록

각 주문 항목은 다음 정보를 포함:

- **메뉴명**: 선택한 메뉴 이름
- **옵션 표시**: 선택한 옵션이 있는 경우 괄호 안에 표시
  - 예: "아메리카노(ICE) (샷 추가)"
- **수량**: X 1, X 2 등
- **가격**: 해당 항목의 총 가격 (단가 × 수량 + 옵션 가격)

**항목 표시 형식**:
```
• 아메리카노(ICE) (샷 추가) X 1
  4,500원
```

**추가 기능** (선택 사항):
- **수량 조절**: +/- 버튼으로 수량 증가/감소
- **삭제 버튼**: 항목 제거 버튼
- **빈 장바구니**: 장바구니가 비어있을 때 안내 메시지 표시

##### 3.3.3.3 총 금액
- **표시 형식**: "총 금액: 12,500원"
- **스타일**: 굵은 글씨, 강조 색상
- **위치**: 항목 목록 하단, 주문하기 버튼 위
- **실시간 업데이트**: 항목 추가/삭제/수량 변경 시 자동 계산

##### 3.3.3.4 주문하기 버튼
- **위치**: 장바구니 영역 하단 중앙
- **스타일**: 주요 액션 버튼 (큰 크기, 브랜드 색상)
- **비활성화 조건**: 
  - 장바구니가 비어있을 때 비활성화
  - 로그인하지 않은 경우 비활성화 또는 로그인 페이지로 이동
- **동작**:
  - 클릭 시: 주문 확인 페이지로 이동 또는 주문 API 호출
  - 성공 시: 주문 완료 페이지로 이동
  - 실패 시: 에러 메시지 표시

#### 3.3.4 디자인 요구사항

- **위치**: 화면 하단 고정 또는 스크롤 가능한 사이드바
- **배경색**: 밝은 회색 또는 흰색
- **테두리**: 둥근 모서리, 그림자 효과 (선택)
- **반응형**: 
  - 데스크톱: 화면 하단 또는 오른쪽 사이드바
  - 모바일: 하단 고정 또는 모달 형태

## 4. 사용자 플로우 (User Flow)

### 4.1 주문 프로세스

```
1. [홈 화면 진입]
   ↓
2. [메뉴 목록 조회]
   ↓
3. [메뉴 선택]
   ↓
4. [옵션 선택 (선택 사항)]
   ↓
5. [담기 버튼 클릭]
   ↓
6. [장바구니에 추가 확인]
   ↓
7. [추가 메뉴 선택 또는 주문하기]
   ↓
8. [주문하기 버튼 클릭]
   ↓
9. [로그인 확인]
   ↓ (로그인 필요 시)
10. [주문 확인 페이지]
   ↓
11. [주문 완료]
```

### 4.2 상세 시나리오

#### 시나리오 1: 첫 주문
1. 사용자가 홈 화면에 접속
2. 메뉴 목록이 자동으로 로드됨
3. 원하는 메뉴 카드에서 옵션 선택 (선택 사항)
4. "담기" 버튼 클릭
5. 장바구니에 항목 추가 확인 (토스트 메시지)
6. 장바구니에서 총 금액 확인
7. "주문하기" 버튼 클릭
8. 로그인 페이지로 이동 (비로그인 시)
9. 로그인 후 주문 확인 페이지로 이동
10. 주문 완료

#### 시나리오 2: 여러 메뉴 주문
1. 첫 번째 메뉴를 장바구니에 추가
2. 두 번째 메뉴를 장바구니에 추가
3. 장바구니에서 각 항목 확인
4. 총 금액 확인
5. "주문하기" 버튼 클릭
6. 주문 완료

## 5. 상태 관리 (State Management)

### 5.1 주요 상태

| 상태 | 설명 | 저장 위치 |
|------|------|----------|
| **메뉴 목록** | 전체 메뉴 데이터 | React State / Context |
| **장바구니** | 선택한 메뉴 및 옵션 | React State / Context / LocalStorage |
| **사용자 정보** | 로그인한 사용자 정보 | Context / LocalStorage |
| **선택된 옵션** | 각 메뉴 카드에서 선택한 옵션 | 컴포넌트 State |

### 5.2 장바구니 데이터 구조

```typescript
interface CartItem {
  id: string;              // 고유 ID
  menuId: string;          // 메뉴 ID
  menuName: string;        // 메뉴명
  basePrice: number;       // 기본 가격
  options: string[];       // 선택한 옵션 목록
  optionPrices: number[];  // 각 옵션의 추가 가격
  quantity: number;        // 수량
  totalPrice: number;      // 총 가격 (단가 + 옵션 가격) × 수량
}

interface Cart {
  items: CartItem[];
  totalAmount: number;     // 전체 총액
}
```

## 6. 인터랙션 디자인 (Interaction Design)

### 6.1 버튼 인터랙션

#### 담기 버튼
- **기본 상태**: 브랜드 색상, 호버 시 약간 어두워짐
- **클릭 시**: 
  - 버튼 애니메이션 (약간 확대 후 원래 크기)
  - 성공 토스트 메시지 표시
  - 장바구니 아이콘 업데이트 (아이템 수 증가)

#### 주문하기 버튼
- **활성 상태**: 브랜드 색상, 클릭 가능
- **비활성 상태**: 회색, 클릭 불가
- **로딩 상태**: 클릭 시 로딩 스피너 표시
- **성공 시**: 주문 완료 페이지로 이동

### 6.2 옵션 선택 인터랙션

- **체크박스 클릭**: 
  - 즉시 체크/언체크
  - 가격 실시간 업데이트
  - 선택된 옵션은 시각적으로 강조

### 6.3 피드백

- **성공 피드백**: 
  - 토스트 메시지 (2-3초 표시)
  - 버튼 애니메이션
- **에러 피드백**: 
  - 에러 메시지 표시
  - 필수 필드 하이라이트
- **로딩 피드백**: 
  - 스켈레톤 UI
  - 로딩 스피너

## 7. 반응형 디자인 (Responsive Design)

### 7.1 브레이크포인트

| 디바이스 | 너비 | 레이아웃 |
|---------|------|---------|
| 모바일 | < 768px | 1열 그리드, 하단 장바구니 |
| 태블릿 | 768px - 1024px | 2열 그리드, 하단 또는 사이드 장바구니 |
| 데스크톱 | > 1024px | 3열 그리드, 하단 또는 사이드 장바구니 |

### 7.2 모바일 최적화

- **터치 친화적**: 버튼 크기 최소 44px × 44px
- **스와이프 제스처**: 메뉴 카드 스와이프 (선택)
- **하단 고정**: 장바구니를 하단에 고정하여 항상 접근 가능
- **간소화된 UI**: 불필요한 요소 제거

## 13. 디자인 가이드라인 (Design Guidelines)

### 13.1 색상

- **브랜드 색상**: COZY 브랜드 색상 (와이어프레임 기준: 다크 그린)
- **주요 액션**: 브랜드 색상
- **텍스트**: 
  - 제목: 다크 그레이 (#333333)
  - 본문: 중간 그레이 (#666666)
  - 보조 텍스트: 밝은 그레이 (#999999)

### 13.2 타이포그래피

- **제목**: 굵은 글씨, 큰 크기
- **본문**: 일반 글씨, 중간 크기
- **가격**: 굵은 글씨, 강조 색상

### 13.3 간격

- **카드 간격**: 16-24px
- **내부 여백**: 16px
- **섹션 간격**: 32-48px

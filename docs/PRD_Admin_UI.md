# OlderBean 관리자 화면 UI/UX PRD
## Admin Interface Product Requirements Document

## 1. 개요 (Overview)

본 문서는 **OlderBean** 커피 주문 서비스의 **관리자 화면(Admin Interface)**에 대한 상세한 UI/UX 요구사항을 정의합니다. 매장 관리자 또는 바리스타가 메뉴 재고를 관리하고, 주문 상태를 실시간으로 확인 및 업데이트할 수 있는 효율적인 인터페이스를 제공하는 것을 목표로 합니다.

### 목표

- **실시간 주문 관리**: 주문 현황을 한눈에 파악하고 빠르게 처리
- **재고 관리**: 메뉴별 재고 수량을 쉽게 조절
- **대시보드**: 주요 통계를 한 화면에서 확인
- **빠른 작업 처리**: 최소 클릭으로 주문 상태 변경

## 3. 상세 컴포넌트 명세

### 3.1 헤더 영역 (Header)

#### 3.1.1 레이아웃

```
┌────────────────────────────────────────────┐
│  COZY    [주문하기]  [관리자] ✓             │
└────────────────────────────────────────────┘
```

#### 3.1.2 구성 요소

| 요소 | 설명 | 스타일 |
|------|------|--------|
| **로고** | "COZY" 텍스트 또는 이미지 로고 | 왼쪽 정렬, 브랜드 색상 |
| **주문하기 버튼** | 사용자 주문 화면으로 이동 | 오른쪽 정렬, 버튼 스타일 |
| **관리자 버튼** | 관리자 화면 (현재 활성화 상태) | 오른쪽 정렬, 활성화 표시 (✓ 또는 하이라이트) |

#### 3.1.3 기능 요구사항

- **로고 클릭**: 홈 화면으로 이동
- **주문하기 버튼**: 
  - 클릭 시 사용자 주문 화면으로 이동
  - 관리자 권한 유지 (세션 유지)
- **관리자 버튼**:
  - 현재 관리자 화면에 있음을 시각적으로 표시
  - 클릭 시 관리자 대시보드로 이동 (현재 페이지 새로고침)

#### 3.1.4 디자인 요구사항

- **높이**: 60-80px
- **배경색**: 흰색 또는 브랜드 색상
- **활성화 표시**: 관리자 버튼에 체크 표시(✓) 또는 배경색 변경
- **반응형**: 모바일에서는 햄버거 메뉴로 변경 가능

### 3.2 관리자 대시보드 (Admin Dashboard)

#### 3.2.1 레이아웃

```
┌──────────────────────────────────────────┐
│ 관리자 대시보드                           │
├──────────────────────────────────────────┤
│ 총 주문: 1    주문 접수: 1    제조 중: 0  │
│ 제조 완료: 0                             │
└──────────────────────────────────────────┘
```

#### 3.2.2 구성 요소

| 요소 | 설명 | 필수 여부 |
|------|------|----------|
| **섹션 제목** | "관리자 대시보드" | 필수 |
| **총 주문 수** | 전체 주문 건수 | 필수 |
| **주문 접수** | 접수 상태인 주문 수 | 필수 |
| **제조 중** | 제조 중인 주문 수 | 필수 |
| **제조 완료** | 완료된 주문 수 | 필수 |

#### 3.2.3 상세 명세

##### 3.2.3.1 통계 카드

각 통계는 카드 형태로 표시되며 다음 정보를 포함:

- **라벨**: 통계 항목명 (예: "총 주문", "주문 접수")
- **값**: 숫자 (예: "1", "0")
- **아이콘**: 시각적 구분을 위한 아이콘 (선택)

**표시 형식**:
```
┌─────────┐  ┌─────────┐  ┌─────────┐  ┌─────────┐
│ 총 주문  │  │ 주문 접수 │  │ 제조 중  │  │ 제조 완료 │
│    1    │  │    1    │  │    0    │  │    0    │
└─────────┘  └─────────┘  └─────────┘  └─────────┘
```

##### 3.2.3.2 실시간 업데이트

- **자동 새로고침**: 주문 상태 변경 시 자동으로 통계 업데이트
- **업데이트 주기**: 5-10초 간격 (설정 가능)
- **시각적 피드백**: 값이 변경될 때 애니메이션 효과 (선택)

#### 3.2.4 디자인 요구사항

- **레이아웃**: 4개 카드를 가로로 배치 (반응형: 모바일에서는 2x2 그리드)
- **카드 스타일**: 
  - 배경색: 흰색 또는 밝은 회색
  - 테두리: 얇은 테두리 또는 그림자
  - 둥근 모서리
- **색상 구분**: 각 통계별로 색상 구분 가능 (선택)
  - 총 주문: 기본 색상
  - 주문 접수: 파란색
  - 제조 중: 주황색
  - 제조 완료: 초록색

### 3.3 재고 현황 (Inventory Status)

#### 3.3.1 레이아웃

```
┌──────────────────────────────────────────┐
│ 재고 현황                                 │
├──────────────────────────────────────────┤
│ ┌─────────┐  ┌─────────┐  ┌─────────┐  │
│ │아메리카노│  │아메리카노│  │카페라떼  │  │
│ │  (ICE) │  │  (HOT)  │  │         │  │
│ │  10개  │  │  10개   │  │  10개   │  │
│ │ [-][+] │  │ [-][+]  │  │ [-][+]  │  │
│ └─────────┘  └─────────┘  └─────────┘  │
└──────────────────────────────────────────┘
```

#### 3.3.2 구성 요소

| 요소 | 설명 | 필수 여부 |
|------|------|----------|
| **섹션 제목** | "재고 현황" | 필수 |
| **메뉴 카드** | 각 메뉴별 재고 정보 | 필수 |
| **메뉴명** | 메뉴 이름 | 필수 |
| **재고 수량** | 현재 재고 수량 | 필수 |
| **수량 조절 버튼** | +/- 버튼 | 필수 |

#### 3.3.3 메뉴 카드 상세 명세

##### 3.3.3.1 메뉴 카드 구성

각 메뉴 카드는 다음 정보를 포함:

- **메뉴명**: 메뉴 이름 (예: "아메리카노 (ICE)")
- **재고 수량**: 현재 재고 수량 (예: "10개")
- **수량 조절 버튼**: 
  - 감소 버튼 (-)
  - 증가 버튼 (+)

##### 3.3.3.2 수량 조절 기능

- **감소 버튼 (-)**:
  - 클릭 시 재고 수량 1 감소
  - 최소값: 0 (음수 불가)
  - 즉시 저장 또는 일괄 저장 (설정 가능)
- **증가 버튼 (+)**:
  - 클릭 시 재고 수량 1 증가
  - 최대값: 제한 없음 (또는 설정 가능)
  - 즉시 저장 또는 일괄 저장 (설정 가능)

##### 3.3.3.3 재고 부족 알림 (선택)

- **재고 부족 기준**: 설정 가능 (예: 5개 이하)
- **시각적 표시**: 
  - 재고 부족 시 경고 색상 (주황색/빨간색)
  - 아이콘 표시 (선택)
- **알림**: 재고가 0이 되면 알림 표시

#### 3.3.4 그리드 레이아웃

- **데스크톱**: 3-4열 그리드
- **태블릿**: 2-3열 그리드
- **모바일**: 1-2열 그리드
- **간격**: 카드 간 적절한 여백 (16-24px)
- **스크롤**: 메뉴가 많을 경우 세로 스크롤

#### 3.3.5 기능 요구사항

- **재고 목록 조회**: 
  - 페이지 로드 시 자동으로 재고 목록 API 호출
  - 로딩 상태 표시
  - 에러 처리
- **수량 조절**:
  - +/- 버튼으로 수량 증가/감소
  - 변경 사항 즉시 반영 (UI 업데이트)
  - 서버에 저장 (즉시 또는 일괄)
- **저장 방식**:
  - **옵션 A**: 즉시 저장 (각 버튼 클릭 시 API 호출)
  - **옵션 B**: 일괄 저장 ("저장" 버튼으로 모든 변경사항 저장)

#### 3.4.1 레이아웃

```
┌──────────────────────────────────────────┐
│ 주문 현황                                 │
├──────────────────────────────────────────┤
│ ┌────────────────────────────────────┐  │

│ │ 아메리카노(ICE) x 1                 │  │

│ └────────────────────────────────────┘  │

│ │ (다른 주문들...)                   │  │
│ └────────────────────────────────────┘  │
└──────────────────────────────────────────┘
```

#### 3.4.2 구성 요소

| 요소 | 설명 | 필수 여부 |
|------|------|----------|
| **섹션 제목** | "주문 현황" | 필수 |
| **주문 카드** | 각 주문에 대한 정보 | 필수 |
| **주문 시간** | 주문 일시 | 필수 |
| **주문 항목** | 주문한 메뉴 및 수량 | 필수 |
| **주문 금액** | 주문 총액 | 필수 |
| **상태 버튼** | 주문 상태 변경 버튼 | 필수 |

#### 3.4.3 주문 카드 상세 명세

##### 3.4.3.1 주문 카드 구성

각 주문 카드는 다음 정보를 포함:

- **주문 시간**: 주문 일시 (예: "7월 31일 13:00")
- **주문 항목**: 
  - 메뉴명 및 옵션
  - 수량
  - 예: "아메리카노(ICE) x 1"
- **주문 금액**: 주문 총액 (예: "4,000원")
- **상태 버튼**: 주문 상태에 따른 액션 버튼

##### 3.4.3.2 주문 상태 및 버튼

주문 상태에 따라 다른 버튼이 표시됩니다:

| 주문 상태 | 표시 버튼 | 동작 |
|----------|----------|------|
| **접수 대기** | "주문 접수" | 접수 상태로 변경 |
| **접수** | "제조 시작" | 제조 중 상태로 변경 |
| **제조 중** | "제조 완료" | 완료 상태로 변경 |
| **완료** | "완료" (비활성) | 더 이상 액션 없음 |

**버튼 스타일**:
- **활성 버튼**: 주요 액션 색상 (브랜드 색상)
- **비활성 버튼**: 회색, 클릭 불가

##### 3.4.3.3 주문 정렬

- **기본 정렬**: 최신 주문이 상단에 표시
- **필터링**: 상태별 필터링 (선택)
  - 전체
  - 접수 대기
  - 접수
  - 제조 중
  - 완료

##### 3.4.3.4 주문 상세 정보 (확장 가능)

주문 카드 클릭 시 상세 정보 표시 (선택):
- 고객 정보
- 주문 옵션 상세
- 주문 메모
- 주문 이력

#### 3.4.5 디자인 요구사항

- **카드 스타일**: 
  - 배경색: 흰색
  - 테두리: 얇은 테두리 또는 그림자
  - 둥근 모서리
- **상태별 색상**: 주문 상태에 따른 색상 구분 (선택)
  - 접수 대기: 파란색
  - 접수: 주황색
  - 제조 중: 노란색
  - 완료: 초록색
- **스크롤**: 주문이 많을 경우 세로 스크롤

## 4. 사용자 플로우 (User Flow)

### 4.1 관리자 작업 프로세스

```
1. [관리자 로그인]
   ↓
2. [관리자 대시보드 진입]
   ↓
3. [주문 현황 확인]
   ↓
4. [주문 상태 업데이트]
   ↓
5. [재고 관리]
   ↓
6. [통계 확인]
```

### 4.2 상세 시나리오

#### 시나리오 1: 주문 접수 및 처리
1. 관리자가 관리자 화면에 접속
2. 대시보드에서 "주문 접수: 1" 확인
3. 주문 현황 섹션에서 새 주문 확인
4. "주문 접수" 버튼 클릭
5. 주문 상태가 "접수"로 변경
6. 대시보드 통계 자동 업데이트
7. "제조 시작" 버튼 클릭
8. 주문 상태가 "제조 중"으로 변경
9. 제조 완료 후 "제조 완료" 버튼 클릭
10. 주문 상태가 "완료"로 변경

#### 시나리오 2: 재고 관리
1. 관리자가 재고 현황 섹션 확인
2. 특정 메뉴의 재고가 부족함을 확인
3. 해당 메뉴의 "+" 버튼 클릭하여 재고 증가
4. 변경 사항 즉시 저장 (또는 일괄 저장)
5. 재고 수량 업데이트 확인

#### 시나리오 3: 통계 모니터링
1. 관리자가 대시보드에서 전체 통계 확인
2. 각 상태별 주문 수 확인
3. 실시간 업데이트 확인
4. 특정 상태의 주문이 많을 경우 대응

## 5. 상태 관리 (State Management)

### 5.1 주요 상태

| 상태 | 설명 | 저장 위치 |
|------|------|----------|
| **주문 목록** | 전체 주문 데이터 | React State / Context |
| **재고 목록** | 메뉴별 재고 데이터 | React State / Context |
| **대시보드 통계** | 주문 통계 데이터 | React State / Context |
| **사용자 정보** | 관리자 정보 | Context / LocalStorage |
| **필터 상태** | 주문 필터 상태 | 컴포넌트 State |

### 5.2 데이터 구조

#### 주문 데이터 구조
```typescript
interface Order {
  id: string;              // 주문 ID
  userId: string;          // 사용자 ID
  status: OrderStatus;     // 주문 상태
  items: OrderItem[];      // 주문 항목
  totalAmount: number;     // 총 금액
  createdAt: string;       // 주문 일시
  updatedAt: string;       // 업데이트 일시
}

enum OrderStatus {
  PENDING = '접수 대기',
  RECEIVED = '접수',
  IN_PROGRESS = '제조 중',
  COMPLETED = '완료',
  CANCELLED = '취소'
}
```

interface Inventory {

## 6. 인터랙션 디자인 (Interaction Design)

### 6.1 버튼 인터랙션

#### 주문 상태 변경 버튼
- **기본 상태**: 브랜드 색상, 클릭 가능
- **클릭 시**: 
  - 로딩 스피너 표시
  - API 호출
  - 성공 시 상태 업데이트 및 통계 갱신
  - 실패 시 에러 메시지 표시

#### 재고 수량 조절 버튼
- **+ 버튼**: 
  - 클릭 시 수량 1 증가
  - 즉시 UI 업데이트
  - 저장 확인 (즉시 또는 일괄)
- **- 버튼**: 
  - 클릭 시 수량 1 감소
  - 최소값(0) 도달 시 비활성화
  - 즉시 UI 업데이트
  - 저장 확인 (즉시 또는 일괄)

### 6.2 실시간 업데이트

- **폴링**: 주문 상태 변경을 주기적으로 확인 (5-10초)
- **WebSocket**: 실시간 업데이트 (선택, 향후 구현)
- **시각적 피드백**: 새 주문 도착 시 알림 (선택)

### 6.3 피드백

- **성공 피드백**: 
  - 토스트 메시지 (2-3초 표시)
  - 상태 변경 애니메이션
- **에러 피드백**: 
  - 에러 메시지 표시
  - 재시도 버튼 제공
- **로딩 피드백**: 
  - 버튼에 로딩 스피너
  - 전체 화면 로딩 (초기 로드 시)
